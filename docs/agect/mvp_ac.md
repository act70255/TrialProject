## C1 目錄樹初始化正確

- 通過條件：
  - 目錄可包含多個檔案（非單一檔案限制）。
  - 目錄可包含子目錄，且子目錄可再包含子目錄（巢狀層級不設上限）。
  - 能建立至少 3 層以上的目錄樹（含根目錄與巢狀子目錄）以驗證遞迴結構。
  - 所有檔案皆隸屬於某個目錄，不存在孤立檔案節點。
  - 結構需與「本次驗收使用的初始化資料」一致（可為題目示例或專案自訂資料集）。
- 必要檢查項目：
  - 單一目錄下是否可同時存在多個檔案。
  - 根目錄下是否同時存在子目錄與檔案節點。
  - 至少一個子目錄下是否再包含子目錄（驗證遞迴樹狀結構）。
  - 深層子目錄（第 3 層以上）是否可被正確建立與存取。
  - 所有檔案是否皆可追溯到其父目錄。
  - 建立順序是否可被後續 DFS 與 Log 重現（同層維持插入順序）。
- 驗收方式：
  - 啟動系統初始化流程後，檢視記憶體物件樹或初始化輸出。
  - 逐項比對節點存在性、父子關係與建立順序。

## C2 目錄樹輸出正確

- 通過條件：
  - Console 能完整輸出目錄樹層級，縮排或前綴可清楚識別父子關係。
  - 檔案列印需包含型別對應細節：Word 頁數、圖片解析度、文字檔編碼、檔案大小。
  - 輸出內容與初始化資料一致，無遺漏或錯誤節點。
- 必要檢查項目：
  - Word 檔案是否顯示頁數與大小。
  - Image 檔案是否顯示解析度與大小。
  - Text 檔案是否顯示編碼與大小。
  - 各節點名稱與屬性值是否與本次驗收資料集一致。
- 驗收方式：
  - 執行目錄樹列印功能，截圖或保存 Console 輸出。
  - 依節點逐行比對層級與檔案細節欄位。

## C3 遞迴總容量計算正確

- 通過條件：
  - 任一目錄總大小計算需包含所有子目錄與檔案（遞迴完整）。
  - 內部計算單位使用 Bytes，且換算遵循 `1KB=1024B`、`1MB=1024KB`。
  - 顯示值與預期人工換算一致，允許顯示格式化但不可改變實際值。
- 必要檢查項目：
  - 目標目錄總大小是否等於其所有子孫檔案大小總和（人工換算比對）。
  - 至少驗證 1 個中介層目錄與 1 個根目錄結果。
  - 若使用題目示例資料，`Root` 預期值為 2,766,324B。
  - 空目錄計算結果是否為 0B。
- 驗收方式：
  - 分別對 Root 與至少兩個子目錄執行總容量計算。
  - 以人工換算結果比對程式輸出（Bytes 與格式化顯示）。

## C4 副檔名搜尋正確

- 通過條件：
  - 輸入副檔名可回傳該目錄樹中所有符合檔案完整路徑。
  - 走訪規則為 DFS Pre-order，且同層以建立順序遍歷。
  - 回傳結果順序與「發現順序」一致，不額外排序。
- 必要檢查項目：
  - 搜尋常見副檔名（如 `.docx`、`.txt`）是否命中正確檔案。
  - 若使用題目示例資料，`.docx` 應命中 2 筆且順序符合 DFS。
  - 搜尋不存在副檔名（如 `.pdf`）時應回傳空集合，不拋出錯誤。
  - 路徑格式是否一致（分隔符、大小寫策略、是否含 Root）。
- 驗收方式：
  - 以至少三組副檔名（命中多筆、命中單筆、無命中）執行搜尋。
  - 比對每組結果內容與順序是否符合預期。

## C5 XML 輸出正確

- 通過條件：
  - 可將當前目錄樹完整序列化為 XML。
  - XML 結構需正確表達目錄階層與檔案語意（型別與屬性資訊）。
  - 驗收以語意一致為主，不要求節點名稱逐字比對題目示例。
- 必要檢查項目：
  - XML 是否保留 Root 到最深層子目錄的巢狀關係。
  - 檔案節點是否包含 Name、Size、CreatedTime 與型別專屬屬性。
  - 三種檔案型別（Word/Image/Text）是否都能正確輸出對應資訊。
  - XML 可被標準解析器成功讀取（格式合法、標籤閉合正確）。
- 驗收方式：
  - 執行 XML 輸出功能並保存輸出檔或字串。
  - 以人工檢視 + XML Parser 驗證結構合法性與欄位完整性。

## C6 Traverse Log 可驗證

- 通過條件：
  - 在「計算總容量」與「副檔名搜尋」兩流程皆輸出 Traverse Log。
  - Log 節點訪問順序與 DFS Pre-order、同層建立順序一致。
  - Log 可追溯每次訪問節點，能對照最終計算或搜尋結果。
- 必要檢查項目：
  - 計算流程是否有 `Visiting: ...` 形式的連續訪問紀錄。
  - 搜尋流程是否有相同走訪規則的訪問紀錄。
  - 兩流程在同一資料樹下的前序訪問順序是否一致。
  - 當資料量增加時，Log 是否仍保持可讀且不遺漏節點。
- 驗收方式：
  - 分別執行計算與搜尋，保存 Console Log。
  - 逐行比對 Log 順序與預期 DFS 順序。

## C7 三種檔案型別與屬性完整

- 通過條件：
  - 系統具備 Word、Image、Text 三種檔案型別建模。
  - 三型別皆包含共通屬性：Name、Size、CreatedTime。
  - 各型別專屬屬性正確：PageCount、Width/Height、Encoding。
- 必要檢查項目：
  - 型別建立後可透過輸出或偵錯觀察到全部屬性值。
  - 型別專屬屬性不會錯置（例如 Text 不應出現 PageCount）。
  - 共通屬性在序列化、搜尋、計算等流程可被一致使用。
  - 新增同型別檔案時不需修改既有核心流程（符合 OCP 趨勢）。
- 驗收方式：
  - 建立三種檔案實例並執行輸出/搜尋/XML/計算流程。
  - 檢查各流程中屬性可見性與使用正確性。
